# Security Policy

## Supported Versions

Security fixes are applied to the latest code on the `main` branch and the most recent tagged release.

## Reporting a Vulnerability

Please do not open public issues for suspected vulnerabilities.

Use GitHub private vulnerability reporting for this repository when possible. If that is unavailable for any reason, contact the maintainer directly and include:

- A concise description of the issue
- Reproduction steps or a proof of concept
- Impact assessment
- Any suggested remediation

I will acknowledge reports as quickly as practical, aim to confirm impact within 72 hours, and coordinate a fix and disclosure timeline before public release.

## Disclosure

After a fix is available, the issue can be disclosed publicly through a GitHub security advisory, release notes, or both, depending on severity and exploitability.
